In Kitchen

In Kitchen is a web application that helps you decide what to do with those leftover ingredients in your fridge. You can list ingredients you have, and it will generate a list of dishes with corresponding recipes that include specified ingredients. Each ingredient you type in will turn into a red tag, which you can get rid of by simply clicking the close button. When ready, click "find recipes!" to have a good meal and clean up your fridge. We first parse the user input string into an array of keywords, and use Yumly API to get recipe data. This was a 4-member team project at HackSC '14, and I worked mostly on the UI, parsing inputs, and integrating parsed output of server calls into HTML contents.

Micromouse

Micromouse is an autonomous maze-solving robot competition. I am currently working with 3 other peers, and we are in charge of the entire design and implementation process. This mouse first explores the maze thoroughly, gathering information about the walls by mainly utilizing IR sensors. Then, with the collected information, it finds a shortest path to the center of the maze. On the hardware level, this involves designing the circuit, mounting and soldering elements, and calibrating sensors. On the software level, we mainly focus on optimizing data structures, bit-level optimization (MCU memory: 64KB), and implementing maze-solving algorithms.

Bug Blast

BugBlast is a maze-based 2D video game. Player can place timer bombs to destroy walls and defeat monsters present in the maze to proceed to the next level with more challenging difficulty. This was a class project in which specifications and basic framework for ticks and graphics were provided. I first organized the class hierarchy following the principles of object-oriented programming, and implemented specifics of each class incrementally from base classes to various derived classes, leveraging inheritance and supporting polymorphism. One of the monster classes had a player-chasing attack mechanism, which I implemnted with a breadth-first search based maze-solving algorithm.

Rocket Launch Dashboard

As part of AIAA (The American Institute of Aeronautics and Astronautics) UCLA branch, I am participating in digitalizing an analog relay system for controls and data acquisition of a rocket launch. Earthquake Locator

Earthquake Locator provides a realtime visualization of earthquake data around the world. The data can be filtered by the time of the earthquake and the magnitude. It is still under development, and I am currently working on the implementation of location based search. The data is provided by USGS, and it is updated every 5 minutes. Movie Database Search Webapp

My partner and I implemented a B+ Tree Index in C++ for a movie database as a class project at UCLA (CS143). It supports loading tuples from input files, select queries on attributes and aggregates with conditions. Insert functions are implemented recursively in order to handle overflows at both leaf and non-leaf levels. It is a generic template that works for any N less than 82 (this limitation arises due to the disk sector size of the simulator we used). It significantly reduced the run-time of queries: for a table of 12,000 tuples, we reduced the disk accesses from 12,000 reads to 5 reads.
Then, we built a web app that supports search and update features to mySQL movie database. It is written in PHP. The video above is a demo of the app.
